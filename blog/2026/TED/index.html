TeD 架构逐步流程与设计意义

完整技术笔记

流程总结

阶段 操作 意义
输入 time-lapse data + rTG 提供原始图像内容 + 时间变化线索
SFE 卷积 → 多尺度空洞卷积 → 拼接压缩 提取多尺度空间结构
TFE rTG 生成权重图 → 调制原始数据 → 多尺度卷积 让时间变化引导特征提取，增强动态建模能力
融合 SFE + TFE 逐元素相加 合并空间与时间特征
DFE 5 个 RSTB 做深层建模 捕获长距离依赖和全局结构
输出 加回浅层特征 + 3 层卷积 恢复细节并生成最终去噪图像

1 输入两路数据：time-lapse data 与 rTG

做什么

输入两种尺寸相同的张量，维度均为 $41 \times 512 \times 512$：

  • time-lapse data：中心帧前后各 20 帧的原始时序图像
  • rTG：相对时间梯度（relative Temporal Gradient），描述相邻帧之间的变化强弱

为什么这么做

这是在给模型提供两类互补信息：

  • time-lapse data 提供原始像素内容，是去噪的主体信息来源
  • rTG 提供时间变化线索，告诉网络哪些地方在动、哪些地方比较稳定

也就是说，模型不是只看”图像长什么样”，还看”图像在时间上怎么变”。这对时序显微图像特别重要，因为噪声通常很强，单帧信息不够稳定，而时序上下文能帮助恢复真实结构。

2 SFE：空间特征提取

做什么

对原始 time-lapse data 做一条空间分支：

  1. 先用一个 Conv2D 把 41 通道映射到 60 通道
  2. 再并行经过 3 个不同 dilation rate 的空洞卷积
  3. 把三路输出在通道维拼接
  4. 再用一次卷积把通道压回 60

为什么这么做

这一步的目标是：从原始图像中提取多尺度空间信息。

先做通道映射：原始输入的 41 帧不能直接当成最终特征来用，所以先把它变成更适合后续学习的特征表示。这相当于先把”原始像素序列”变成”初步特征图”。

用不同 dilation rate 的卷积：不同膨胀率的卷积看到的上下文范围不同：

  • 小 dilation：看局部细节
  • 中 dilation：看中尺度结构
  • 大 dilation：看更大范围的空间上下文

如果只用普通卷积，感受野有限；空洞卷积可以扩大视野，又不显著增加参数量。

拼接后再压回 60 通道：三路卷积提取的是不同尺度的信息，拼接以后就把这些信息合在一起。再压回 60 通道，是为了统一特征维度、控制计算量，方便后面与 TFE 输出相加融合。

本质意义

SFE 的作用是：从原始图像中提取”空间上可靠”的多尺度结构信息。它主要解决的问题是：”这个地方在空间上像不像真实结构？”

3 TFE：时间特征提取与 TGAM

做什么

TFE 里最关键的是 TGAM（Temporal Gradient Attention Module）：

  1. rTG 经过卷积
  2. 再经过 Sigmoid，得到 $[0, 1]$ 的权重图
  3. 这个权重图与原始 time-lapse data 做逐元素相乘
  4. 得到的加权数据再进入和 SFE 类似的卷积 + 空洞卷积 + 拼接 + 压缩流程

为什么这么做

rTG 先变成权重，不是直接拿来分类：rTG 本身描述的是”变化强度”，但它还不是最终可直接用于恢复的特征。所以先通过卷积和 Sigmoid，把它变成一张软注意力图。这张图告诉网络：

  • 哪些区域时间变化剧烈
  • 哪些区域时间变化较稳定
  • 哪些位置更值得关注、哪些位置应该抑制

为什么要和原始 time-lapse data 相乘：这个操作的核心意义是：用时间变化信息去调制原始图像。也就是说：

  • 静态、稳定区域：保留更多信息
  • 动态剧烈区域：降低其影响，避免错误放大不稳定变化

这一步不是为了”创造新信息”，而是为了重加权已有信息，让后面的特征提取更聚焦、更鲁棒。

为什么后面还要再做卷积和拼接：注意力本身只是在说”哪里重要”，但它不擅长真正提炼结构。后面的卷积负责把被重加权的数据进一步变成可用特征，提取局部时间模式、变化区域的空间结构、多尺度上下文。

本质意义

TFE 的作用是：把时间变化信息变成一个引导机制，帮助模型更聪明地利用时序数据。它主要回答的问题是：”哪些地方的时间信息更值得信任？”

4 SFE + TFE 融合：相加而不是拼接

做什么

将 SFE 和 TFE 的输出做 element-wise addition，得到融合后的浅层特征，再送进 DFE。

为什么这么做

SFE 和 TFE 是两种互补特征：

  • SFE 提供空间结构信息
  • TFE 提供时间引导后的特征信息

做相加而不是拼接，有几个考虑：

  1. 保持通道规模稳定：相加不会增加通道数，维度简单，计算更轻。
  2. 强调互补融合：相加相当于把两种来源的特征直接叠加，让网络在同一特征空间里综合判断。
  3. 让后续深层模块更专注于高层建模：如果在这里就拼接，通道会翻倍，后续 DFE 计算负担更大。相加是更轻量的融合方式。

本质意义

这是在告诉模型：空间信息和时间信息都重要，先把它们合并成一个统一的中间表示。

5 DFE：用 Swin Transformer 做深层特征建模

做什么

DFE 由 5 个 RSTB（Residual Swin Transformer Block）组成。每个 RSTB 内部大致是：

  1. 把输入切成 patch
  2. 展平并加入位置信息
  3. 送入多头自注意力的 Swin Transformer
  4. 再还原回二维空间
  5. 接卷积层

为什么这么做

为什么要用 Transformer：卷积擅长局部结构，但对长距离依赖建模能力有限。而时序显微图像中，很多重要结构具有非局部关联：

  • 远处的区域可能共同构成同一个细胞结构
  • 某些模式需要跨较大空间范围才能识别

Transformer 的自注意力机制适合做这种全局关系建模。

为什么用 Swin Transformer 而不是普通 Transformer：普通 Transformer 计算太贵，尤其在高分辨率图像上。Swin Transformer 用局部窗口注意力，既能建模较强的上下文关系，又更节省计算。它的意义是：

  • 保留全局建模能力
  • 控制复杂度
  • 更适合大尺寸图像

为什么还要和卷积结合：

  • Transformer 负责”远处之间有什么关系”
  • 卷积负责”局部纹理和边缘怎么恢复”

这对去噪任务非常重要，因为去噪既要保结构，也要保细节。

本质意义

DFE 负责：在融合后的特征基础上，进行深层全局推理，恢复更完整、更一致的图像结构。

6 RSTB 内部的 Patch 切分和窗口注意力

做什么

每个 RSTB 里，输入先切成大小为 $4 \times 4$ 的 patch，然后：

  1. 展平
  2. 加位置编码
  3. 做 6 头窗口注意力，窗口大小为 8
  4. 再还原回空间结构
  5. 接卷积

为什么这么做

  • Patch 切分：把图像分成小块后，模型更容易处理，也更适合注意力机制。
  • 位置编码：注意力本身不天然知道顺序或位置，所以要告诉它每个 patch 在图像中的位置。
  • 窗口注意力：只在局部窗口内做注意力，避免全局注意力的高计算量。
  • 后接卷积：注意力后再用卷积补充局部空间归纳偏置，让特征更加稳定。

本质意义

它是在一个可控计算成本下，让模型既能看局部，也能看较远范围。

7 最后输出重建：残差加回浅层特征

做什么

DFE 输出后，再把前面的 SFE + TFE 浅层融合特征加回来，然后经过 3 层卷积得到最终去噪结果。

为什么这么做

为什么加回浅层特征：浅层特征里包含很多细节、边缘、纹理等信息。如果完全依赖深层模块，容易丢失这些细节。把浅层特征加回来，相当于给最终输出留一个”细节保底”。

为什么还要经过三层卷积：最后几层卷积负责把融合后的特征变成最终图像输出，完成：

  • 通道压缩
  • 细节整合
  • 噪声抑制
  • 图像重建

本质意义

这是在实现：既有深层推理结果，又保留早期细节信息，最后生成更干净、更完整的图像。
